## 1. 발견한 DART 복잡성

### 1.1 계정과목 다양성

| 회사 | 매출 계정과목 | 비고 |
|------|-------------|------|
| YBM넷 | "매출액" | 일반적 |
| **삼성전자** | **"영업수익"** | 특수 |
| LG전자 | "매출액" | 일반적 |
| 기타 | "순매출액", "수익(매출액)" 등 | 다양 |

**문제**: 키워드 "매출액"만으로는 삼성전자 등 커버 불가

### 1.2 thstrm_nm (당기 이름) 다양성

```
발견된 패턴:
- "당기"
- "제 24 기"
- "2023년"
- (빈 문자열)
```

**문제**: "당기" 조건으로는 많은 기업 누락

### 1.3 단위 이슈

```
DART API 응답:
- thstrm_amount: 숫자 (원 단위)

v1 초기 구현:
- * 1000 (천원 → 원 변환) ← 잘못됨!

실제:
- 이미 원 단위 ✅
```

## 2. v9 개선 구현

### 2.1 우선순위 매칭 (핵심!)

```python
# Before (v1 초기)
revenue_items = [
    item for item in financials
    if '매출액' in item.get('account_nm', '')
]
# → 삼성전자 실패 ❌

# After (v1 개선)
revenue_keywords = [
    ['매출액'],      # 1순위
    ['영업수익'],    # 2순위 (삼성전자 등)
    ['수익', '매출'], # 3순위
]

for keywords in revenue_keywords:
    items = [
        item for item in financials
        if all(kw in item.get('account_nm', '') for kw in keywords)
    ]
    if items:
        revenue_items = items
        break
# → 삼성전자 성공 ✅
```

### 2.2 조건 제거

```python
# Before
if '당기' in item.get('thstrm_nm', ''):
    ...
# → 많은 기업 누락

# After
# thstrm_nm 조건 완전 제거
# → 모든 기업 커버
```

### 2.3 단위 수정

```python
# Before
revenue = float(revenue_raw) * 1000  # 천원 → 원
# → 60조원 (잘못됨)

# After
revenue = float(revenue_raw)  # 이미 원 단위
# → 604.8억원 (정확)
```
